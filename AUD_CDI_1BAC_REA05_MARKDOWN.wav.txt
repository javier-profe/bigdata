Vamos allá. El elemento más básico que seguramente
quieras comenzar a crear en Markdown son los encabezados. Existen hasta 6 niveles, y para
generarlos no tienes más que añadir una almohadilla al comienzo de la palabra o frase.
Por ejemplo, una almohadilla generará un encabezado de tipo H1. Dos almohadillas generarán
un encabezado de tipo H2, y así hasta 6. Como ves, no hace falta que cierres la palabra
o frase con las mismas almohadillas, aunque podrías hacerlo por estética, yo no te lo
recomiendo ya que en algunas aplicaciones esto da problemas.
Para añadir citas a tus textos, tan solo tienes que incluir el símbolo mayor que antes
de las mismas, tal que así. Si quieres crear una cita con varios altos de línea, no tienes
más que añadir el mismo símbolo en cada una de ellas.
Las listas desordenadas que tanto cuesta generar con HTML no te llevarán nada de trabajo con
Markdown, ya que solo tienes que comenzarlas con un guión para que se detecten automáticamente.
Algunas aplicaciones aceptarán también asteriscos y símbolos de más para este fin.
Si lo que quieres crear es una lista numerada, lo único que tienes que hacer es incluir
el número seguido de un punto y un espacio antes de comenzar a escribir. La lista numerada
se detectará automáticamente. Puedes hacer lo mismo para el resto de elementos.
Si quieres, puedes combinar estas listas y anidarlas, para lo cual solamente tienes
que incluir un espacio utilizando la tecla tabulador.
Para incluir separaciones entre secciones de texto, basta con que escribas tres guiones
bajos seguidos. Markdown intercambiará estos símbolos por la etiqueta HR.
Como estás viendo, Markdown es muy simple e intuitivo. Ahora voy a enseñarte dos elementos
que seguramente sean básicos para tus escritos, las negritas y las cursivas.
Markdown admite dos maneras de crear estos estilos, mediante asteriscos o mediante guiones
bajos. Yo te recomiendo utilizar siempre asteriscos.
Si rodeas una palabra o frase con tan solo uno de estos elementos, por ejemplo el asterisco,
aplicarás una cursiva al texto en cuestión. Si en vez de uno utilizas dos, se aplicará
la famosa negrita o resaltado. Y como estarás imaginando, si quieres realizar
ambas cosas, no tienes más que utilizar los tres elementos a la vez. De esta forma tendrías
una palabra o frase en negrita y en cursiva. Para crear enlaces, no tienes más que añadir
entre corchetes la palabra o frase que quieras utilizar como texto ancla, en mi caso, EnfoqueNomada,
y acto seguido y entre paréntesis, el enlace en cuestión, tal que así.
Puedes añadir información esta entrecomillada justo después del enlace, todavía entre
los paréntesis, para que esto sea el title del enlace, tal como estás viendo en pantalla.
Si quieres que la URL sea el propio enlace, no tienes por qué escribirla dos veces entre
los corchetes y después entre los paréntesis. Basta con que incluyas la propia URL entre
los símbolos menor que y mayor que. Las imágenes se añaden de una manera similar
a los enlaces. En este caso, lo que tienes que hacer es escribir el texto alternativo
asociado a las mismas entre corchetes y acto seguido y entre paréntesis la ruta de la
imagen. Como ves, este método es idéntico al que acabo de enseñarte para los enlaces.
Para que verdaderamente funcione como código para insertar imágenes, no tienes más que
añadir un símbolo de exclamación al principio del mismo.
Si escribes documentación técnica con Markdown, también puedes incluir cajetines de código
de cualquier lenguaje de programación para que este no sea interpretado y se muestre
de manera limpia. Si solo vas a incluir una palabra o frase, bastará con que la inicies
con cuatro espacios en blanco, tal que así. Pero si vas a generar mucho más código,
es más cómodo que simplemente lo encierres entre tres virgulillas por arriba y tres por
abajo, de esta forma. Y si solo quieres resaltar código dentro de una frase, lo único que
tienes que hacer es incluirlo entre dos acentos graves, aunque en este caso la previsualización
nos aprecia muy bien. Como ves, Markdown utiliza símbolos comunes
para marcar el texto de una manera muy rápida, así que es probable que ahora mismo te estés
preguntando si no vas a poder utilizar dichos símbolos de manera normal. La respuesta es
que sí, pero antes de los mismos tendrás que añadir una barra invertida, tal que así.
De esta manera, si te fijas en la previsualización, esta dejará de estar en cursiva para mostrar
los símbolos de Markdown. De esta forma, podrás seguir utilizando esos símbolos
siempre que lo necesites. Con todo esto que has visto, ya estás listo
para comenzar a utilizar Markdown en tu blog y en tu vida diaria.
Recuerda que si quieres conocer cuáles son las mejores aplicaciones para sacarle el máximo
potencial a este lenguaje, o si quieres aprender Markdown a nivel extremo creando tablas o
